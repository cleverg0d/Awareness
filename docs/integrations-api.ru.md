# API интеграций

[English](integrations-api.md)

Небольшой API, который позволяет доверенному внешнему сервису назначать сотруднику обучение по
информационной безопасности. Рассчитан на инструменты вроде аудитора парольной политики, который
хочет записывать сотрудников, не прошедших проверку, на подходящий курс. Это API для взаимодействия
между сервисами, авторизованное токеном, отдельное от обычного пользовательского входа по сессии, и
вызывать его из браузера не предполагается.

## Получение токена

Администратор создает токен в консоли, в разделе `/console/integrations`, дает ему название и
выбирает, какие курсы он может назначать. Токен показывается один раз, в момент создания, поэтому
сохраните его в надежном месте, например в секретном хранилище вашего сервиса. Если он утечет,
ущерб ограничен только теми курсами, на которые он был настроен: попросите администратора отозвать
токен и выпустить новый.

## Аутентификация

Передавайте токен как bearer-токен в каждом запросе:

```
Authorization: Bearer awr_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## Эндпоинты

Базовый адрес: `https://<ваш-хост-awareness>/api/integrations/v1/`.

`GET courses/` возвращает список курсов, которые разрешено назначать этому токену, что удобно
использовать вместо жестко прописанного идентификатора курса. Ответ выглядит как массив объектов
вида `{ "id": 4, "title": "Памятка по парольной защите" }`.

`POST assign-training/` назначает сотруднику обучение по курсу и идемпотентен: повторный вызов для
того же сотрудника и курса не создаст дубль назначения. Тело запроса выглядит как
`{ "employee_email": "employee@company.local", "course": 4, "reason": "10+ слабых паролей найдено
при последнем аудите" }`, где `reason` - необязательный свободный текст, который сохраняется в
журнале для аудита и никогда не показывается сотруднику. Успешный ответ возвращает статус 201, если
назначение создано впервые, либо 200, если сотрудник уже был назначен, с телом вида
`{ "wave_id": 12, "wave_name": "Автоматическое назначение: Памятка по парольной защите",
"assignment_id": 87, "created": true }`.

Ответ 401 означает, что токен отсутствует, неверен или отозван. Ответ 404 намеренно объединяет три
разные ситуации: сотрудник не найден или неактивен, курс не найден или неактивен, либо курс просто
не входит в список разрешенных для этого токена. Эти случаи специально неотличимы друг от друга,
чтобы утекший токен нельзя было использовать для перебора email сотрудников или для обнаружения
курсов за пределами своей области действия; точная причина при этом остается видна администратору
в журнале интеграций в консоли. Ответ 429 означает превышение лимита запросов.

## Лимиты запросов

По умолчанию действует лимит 60 запросов в час на токен. Если для легитимного массового сценария
нужен более высокий лимит, обратитесь к администратору: это область троттлинга Django REST
Framework с именем `integration`, ее можно поменять в `backend/config/settings.py`.

## Аудит

Каждый вызов, успешный или нет, записывается в журнал вместе с использованным токеном, сотрудником,
курсом и результатом. Администраторы видят этот журнал в консоли рядом со списком токенов.
