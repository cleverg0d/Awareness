# История изменений

[English](CHANGELOG.md)

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), проект следует
[Semantic Versioning](https://semver.org/lang/ru/): MAJOR версия - несовместимые изменения (нужны
ручные шаги при обновлении), MINOR - новые возможности без потери совместимости, PATCH - только
исправления. Текущая версия отображается внизу меню консоли и в шапке Django Admin, а также
доступна по `GET /api/health/`. Перед обновлением на новую версию всегда смотрите раздел
[docs/upgrading.ru.md](docs/upgrading.ru.md) и секцию "Важно при обновлении" ниже, если она есть у
нужной версии.

Каждый пункт помечен значком: ✚ - добавлена возможность, ✓ - что-то исправлено или улучшено.
Отдельно про безопасность здесь не пишем даже в общих словах - подробности закрытых уязвимостей не
публикуются, чтобы не подсказывать, где искать проблемы тем, кто не обновился.

## [Unreleased]

Пока нет незапланированных изменений.

## [1.3.5-pro] - 2026-07-13

✓ Страница настроек сотрудника переоформлена - добавлена карточка личных данных (ФИО, роль,
email, отдел, должность), карточка темы оформления стала компактной вместо непропорционально
крупной, форма смены пароля стала компактнее.

## [1.3.4-pro] - 2026-07-13

✓ Ограничен размер загружаемых картинок - иконки наград и изображения в редакторе глав курса.

## [1.3.3-pro] - 2026-07-13

✓ Интерфейс консоли стал адаптивным - таблицы скроллятся по горизонтали на узких экранах, формы
перестраиваются в один столбец.

✚ Боковое меню консоли на мобильном - постоянная узкая полоска с иконками вместо выезжающей
панели, с ручным сворачиванием на десктопе.

## [1.3.2-pro] - 2026-07-13

✚ "Волны обучения", "Курсы", "Награды" и "Рейтинг" объединены в один пункт меню "Обучение" с
вкладками - раньше это были 4 отдельных пункта в сайдбаре про, по сути, одну и ту же работу.

✚ На дашборде появился фильтр по периоду и более наглядные цветные плитки со статусами.

## [1.3.1-pro] - 2026-07-13

✚ Рейтинг сотрудников по проценту сданных курсов - по компании и по отделу. По умолчанию
выключен, т.к. это HR-чувствительная тема; при включении сотрудники видят топ-10 и свое место, не
полный список.

## [1.3.0-pro] - 2026-07-13

✚ Награды за прохождение курсов (ранее "значки") - админ настраивает картинку и условие
получения: за конкретный курс, за любой курс или за конкретную волну (для ежегодных/квартальных
обязательных курсов у каждого цикла может быть своя награда, без задваивания при честной
пересдаче). У каждого сотрудника есть публичная страница-подтверждение полученной награды.

## [1.2.5-pro] - 2026-07-13

✚ Контроль фокуса на тесте - переключение на другое окно или вкладку во время прохождения теста
фиксируется.

## [1.2.4] - 2026-07-08

✓ `manage.py createsuperuser` падал с `TypeError: ... missing 1 required positional argument:
'username'` на чистой базе - исправлено. В документации также поправлены упоминания раздела
консоли для переключателя блокировки входа и журнала - теперь "Безопасность" и "Журналы" вместо
устаревшего "Интеграции".

## [1.2.2] - 2026-07-08

✓ Защита входа (блокировка аккаунта) вынесена из "Интеграций" в отдельный раздел "Безопасность" -
это не внешняя интеграция, и теперь есть задел под другие настройки безопасности в будущем.

## [1.2.1] - 2026-07-08

✓ Журналы входов, вызовов интеграций и отправок уведомлений были разбросаны по трем разным
страницам консоли - теперь собраны вместе в разделе "Журналы", настройки рядом с ними остались
на прежних местах.

✓ Инструкция по обновлению описывала только путь через Docker - добавлены аналогичные шаги для
ручной установки (без Docker).

## [1.2.0] - 2026-07-08

✚ Новую блокировку аккаунта можно выключить в разделе "Интеграции → Защита входа", если она
мешает - ограничение по IP на страницу входа при этом остается.

✚ Журнал входов теперь фиксирует каждую попытку, успешную и нет, с введенным email и IP-адресом,
доступен там же.

## [1.1.0] - 2026-07-08

✚ Вход теперь защищен от подбора пароля не только по IP, но и на уровне конкретного аккаунта:
после 5 неверных паролей подряд аккаунт блокируется на 15 минут независимо от того, с какого IP
пришла следующая попытка. Сброс пароля сотруднику из консоли заодно снимает активную блокировку.

## [1.0.1] - 2026-07-07

✓ Из записи о версии 1.0.0 в этом файле убраны технические детали закрытых до релиза багов и
уязвимостей, оформление списка приведено к единому виду.

## [1.0.0] - 2026-07-07

Первый релиз, подготовленный для реальной эксплуатации и публикации в открытый доступ.

✚ Курсы из глав с полноценным WYSIWYG-редактором (текст, картинки, списки, ссылки), вопросы с
одним или несколькими правильными вариантами ответа.

✚ Волны обучения: назначение конкретным сотрудникам, отделу или всем сразу, дедлайн, лимит попыток,
запуск кампании в один клик со страницы курса.

✚ Дашборд для ИБ-менеджера: статусы прохождения, разбивка по отделам, распределение баллов,
отдельный список проблемных сотрудников, экспорт результатов в CSV.

✚ Вход через Active Directory / LDAP, настраивается прямо в консоли без правки конфигов.

✚ Роли: сотрудник, менеджер обучения, администратор - переиспользуют штатные права Django, роль
можно сменить самостоятельно в разделе "Сотрудники", без доступа к базе или Django Admin.

✚ API-токены для внешних интеграций (например, аудитора парольной политики), привязанные к
конкретным курсам при создании.

✚ Уведомления по Email (SMTP), Telegram, Slack и Microsoft Teams с шифрованием секретов каналов в
базе.

✚ Двуязычный интерфейс, переключение между русским и английским на лету.

✚ Команда генерации демо-данных для проверки портала на пустой базе без реальных сотрудников.
